## Release-Blocking Criteria

For this documentation gate, P0 release-blocking checks are the flows currently
accepted for the contact-host documentation slice: contact CTA rendering,
contact start, duplicate prevention, message send, `/api/messages`, listing
contactability, focused Chromium listing-detail contact-first behavior,
Chromium messaging journey, Mobile Chrome no-deps inbox/thread behavior, and
setup-backed Mobile Chrome inbox/thread behavior. These have current passing
evidence.

Full viewer-state route contract tests, checkout browser return tests,
realtime/RLS proof, suspended/paywall-required/unavailable listing-detail state
matrix, and the full Firefox/WebKit browser matrix remain P1 because they are
clearly marked as unverified and the docs do not claim those behaviors are
runtime-proven. Promote any of these to P0 if the next release requires paid
unlock, direct HTTP contract proof, realtime delivery, or full-matrix
mobile/browser coverage as launch blockers.
