## Cross-State Transitions

| Transition | Trigger | State change | Failure behavior | Release evidence / gap |
|---|---|---|---|---|
| Filter/sort/location apply | Search controls write URL params | Pending local draft becomes committed URL state; parser rebuilds server/client search state | Invalid values are normalized/dropped by parser; broader adversarial cases remain confidence-building coverage | Focused filter, pagination, and URL-state specs passed; broader root specs not run |
| Load more | `nextCursor` exists and user clicks load more | `ClientResults` enters load-more pending, appends unique results or records error | Invalid cursor/rate-limit/V2 unavailable paths return safe empty/error state and keep existing list | `search-pagination.spec.ts` passed |
| Map pan/zoom | User moves map | Viewport and active pan bounds update; wrapper fetches or reuses map data | Missing bounds enters `location-required`; map API/style/tile failure shows map error/a11y states, with V1-only API mock cases still skipped in V2 mode | `search-map-desktop.spec.ts`, `search-map-mobile.spec.ts`, and `map-errors-a11y.anon.spec.ts` passed |
| Listing card image failure | Image `onError` fires | Failed image index is removed from `displayImages`; all failed/empty images switch to placeholder state | Card still links to detail and shows "No Photos"; no listing-card-specific E2E failure spec was run | Source evidence only; listed as confidence-building test gap |
| Favorite unauthenticated | Anonymous user clicks save | Optimistic state reverts and client navigates to `/login` | Other favorite API errors revert and show toast | Anonymous and authenticated saved-listing specs passed; component CSRF/header tests not run |
| Save search submit | User submits saved-search modal | Modal/action enters pending, then success or error | Unauthorized/paywall branch remains partially traced and should stay documented as a P1 gap | Saved-search specs not run |
| Mobile map/list toggle | Mobile user toggles or sheet state changes | Preference/context switches list/map/sheet/filter state | Failure-mocked mobile map and mobile URL-state cross-checks remain unverified | Mobile map/list spec passed; broader mobile tools not run |

The expanded state-machine diagram is `diagrams/state-machine-search-url-map.mmd`. It is intentionally a documentation model, not executable state-machine source.

Remaining verification gaps: V1-only map API mock cases, broader non-gate E2E coverage, broader map/list synchronization coverage, save-search auth/paywall behavior, listing-card-specific image/navigation failure E2E, and stale map/list data prevention. Evidence: `unknowns.md` G001, G006, G011; `runtime-verification.md`.
