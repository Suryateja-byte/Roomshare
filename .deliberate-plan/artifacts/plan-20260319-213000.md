## Fix 4: MessageList API Shape Mismatch + Error Handling (REDESIGNED)

**Priority**: P1 | **Risk**: LOW-MEDIUM | **Effort**: 15 min | **Confidence**: 96%

REDESIGNED based on harsh-critic blocker: original `data.conversations ?? data` crashes on error responses.

**File**: `src/components/MessageList.tsx`

**Before** (lines 26-45):
```typescript
  const [conversations, setConversations] = useState<Conversation[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch("/api/messages")
      .then((res) => res.json())
      .then((data) => {
        setConversations(data);
        setLoading(false);
      })
      .catch((err) => {
        console.error(err);
        setLoading(false);
      });
  }, []);

  if (loading) return <div className="p-4">Loading conversations...</div>;

  if (conversations.length === 0) {
```

**After**:
```typescript
  const [conversations, setConversations] = useState<Conversation[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(false);

  useEffect(() => {
    fetch("/api/messages")
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then((data) => {
        const items = Array.isArray(data.conversations) ? data.conversations : [];
        setConversations(items);
        setLoading(false);
      })
      .catch((err) => {
        console.error(err);
        setError(true);
        setLoading(false);
      });
  }, []);

  if (loading) return <div className="p-4">Loading conversations...</div>;

  if (error) {
    return <div className="p-4 text-destructive">Failed to load messages. Please try again.</div>;
  }

  if (conversations.length === 0) {
```

**Why Array.isArray not ?? fallback**:
- Harsh-critic blocker: `?? data` falls through to raw object on error responses
- Array.isArray guarantees type safety
- !res.ok catches HTTP errors before JSON parsing
- Error state UI shows meaningful message instead of silent "No messages"

**Test plan**:

2. Mock API 401 -> error state shown
3. Mock API 500 -> error state shown

5. E2E: messages page with real conversations -> verify visible

**Rollback**: Revert component. No DB impact.

```typescript
function isAuthorized(request: NextRequest): boolean {
  const secret = process.env.E2E_TEST_SECRET;
  if (!secret || secret.length < 16) return false;
  const authHeader = request.headers.get("authorization");
  if (!authHeader?.startsWith("Bearer ")) return false;
  const token = authHeader.slice(7);
  if (token.length !== secret.length) return false;
  const encoder = new TextEncoder();
  const a = encoder.encode(token);
  const b = encoder.encode(secret);
  return crypto.subtle.timingSafeEqual(a, b);
}
```
